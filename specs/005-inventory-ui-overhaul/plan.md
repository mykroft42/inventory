# Implementation Plan: Inventory UI Overhaul

**Branch**: `005-inventory-ui-overhaul` | **Date**: 2026-04-21 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-inventory-ui-overhaul/spec.md`

## Summary

Replace the card-based inventory grid with a compact shadcn/ui `Table` (32–36px rows) and migrate the build toolchain from Create React App to Vite to support Tailwind CSS v3 and shadcn/ui. Apply the new component library consistently across all existing pages and components. All existing functionality (inline quantity +/−, direct quantity editing, delete with undo toast, form validation) is preserved.

## Technical Context

**Language/Version**: TypeScript 4.9.5 / React 19.2.5
**Primary Dependencies**: Vite 5.x (replacing react-scripts 5), Tailwind CSS 3.4.x, shadcn/ui (component registry), `tailwind-merge`, `tailwindcss-animate`, `class-variance-authority`, `clsx`, Sonner (toast)
**Storage**: N/A — frontend-only change; backend SQLite via EF Core is unchanged
**Testing**: Jest + React Testing Library (frontend unit), Cypress (e2e)
**Target Platform**: Web browser, mobile-first (Tailwind `sm:` breakpoint at 640px)
**Project Type**: Web application — React SPA frontend + ASP.NET Core REST API backend
**Performance Goals**: Table with 100 items renders without layout degradation; quantity updates reflected in UI within 1 second
**Constraints**: No backend changes; existing REST API contract unchanged; `REACT_APP_API_URL` env var renamed to `VITE_API_URL` post-migration
**Scale/Scope**: ≤100 inventory items; 5 existing components + 2 new components (`InventoryTable`, `QuantityCell`)

# Implementation Plan: Edit Item Fields on Detail Page

**Branch**: `007-edit-item-fields` | **Date**: 2026-04-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/007-edit-item-fields/spec.md`

## Summary

Add always-visible category dropdown and expiration date picker controls to the Item Detail Page, with a Save button that persists both fields via the existing `PUT /api/inventory/{id}` endpoint. A backend model validator that incorrectly rejects past expiration dates is removed as a prerequisite, unblocking both this feature and existing items with past dates from being updated.

## Technical Context

**Language/Version**: TypeScript (React 18) — frontend; C# / .NET 10 (ASP.NET Core) — backend
**Primary Dependencies**: React Router, shadcn/ui (Select, Input, Button), Vitest + React Testing Library (frontend); xUnit, WebApplicationFactory (backend)
**Storage**: Existing database via Entity Framework Core — no schema changes
**Testing**: Vitest + React Testing Library (frontend); xUnit integration tests (backend)
**Target Platform**: Web application (mobile-first, WCAG 2.1 AA)
**Project Type**: Web application (frontend + backend)
**Performance Goals**: Standard web app — save round-trip completes within normal network latency; no special targets
**Constraints**: Mobile-first layout; touch targets ≥ 44×44 px; no additional third-party libraries
**Scale/Scope**: Single-user household app; no concurrency concerns

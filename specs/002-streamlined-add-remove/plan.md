# Implementation Plan: Streamlined Inventory Add/Remove

**Branch**: `002-streamlined-add-remove` | **Date**: 2026-04-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-streamlined-add-remove/spec.md`

## Summary

Replace the existing add-item form with a streamlined combo-box interface that autocompletes from existing item names and increments quantity when an existing item is matched. Add single-click removal with a 5-second undo toast. Implement soft-delete at the data layer to satisfy Constitution Principle III while presenting deletion as permanent to the user after the undo window.

## Technical Context

**Language/Version**: C# / .NET 10 (backend); TypeScript / React 19 (frontend)
**Primary Dependencies**: ASP.NET Core, Entity Framework Core + SQLite, React Testing Library, Jest, xUnit
**Storage**: SQLite via EF Core; migration adds `DeletedAt` column and nullable `Category`
**Testing**: xUnit + WebApplicationFactory (.NET); Jest + React Testing Library (React)
**Target Platform**: Local web app (localhost); desktop + mobile browsers
**Project Type**: Web application (React SPA + ASP.NET Core REST API)
**Performance Goals**: Autocomplete suggestion list appears within 300ms of first keystroke (client-side filtering — effectively instant)
**Constraints**: Single-user; no auth; max ~500 items
**Scale/Scope**: One active user; ~500 inventory items; ~10 interactions/day

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|--------------------------------------|
| Principle III: spec says "permanent delete" but constitution requires soft-delete | Undo feature requires the record to survive the 5-second window; audit integrity requires the record exist post-delete | Hard-delete with client-side re-create on undo loses the original record ID and audit history |
| Name-only uniqueness (change from name+category) | Autocomplete increment requires 1:1 name→item mapping; ambiguous name resolution would require extra UI steps violating SC-001 | Disambiguation prompt would add an interaction, violating the ≤3 interactions success criterion |
